# Git Workflow Guide

## Branch Structure

- **main**: Production-ready code
- **develop**: Development branch where features are merged
- **version-X.X**: Release branches (e.g., version-1.0, version-1.1)

## Initial Setup

Run these commands to set up the repository:

```bash
# Add remote (if not already added)
git remote add origin https://github.com/davidsimo2105/wedding-invitation-panorama.git

# Push main branch
git checkout main
git push -u origin main

# Push develop branch
git checkout develop
git push -u origin develop
```

## Workflow for Releases

### Creating a Version Branch

1. **Create a new version branch from develop:**
   ```bash
   git checkout develop
   git pull origin develop
   git checkout -b version-1.0
   ```

2. **Make your changes and commit:**
   ```bash
   git add .
   git commit -m "Release version 1.0: [description of changes]"
   ```

3. **Push the version branch:**
   ```bash
   git push -u origin version-1.0
   ```

### Merging Version Branch to Develop (if approved)

1. **Switch to develop:**
   ```bash
   git checkout develop
   git pull origin develop
   ```

2. **Merge the version branch:**
   ```bash
   git merge version-1.0
   ```

3. **Push develop:**
   ```bash
   git push origin develop
   ```

### Merging Develop to Main (for production releases)

1. **Switch to main:**
   ```bash
   git checkout main
   git pull origin main
   ```

2. **Merge develop:**
   ```bash
   git merge develop
   ```

3. **Push main:**
   ```bash
   git push origin main
   ```

## Example Workflow

```bash
# Start a new release
git checkout develop
git pull origin develop
git checkout -b version-1.0

# Make changes, commit, push
git add .
git commit -m "Release v1.0: Initial wedding invitation"
git push -u origin version-1.0

# After testing, merge to develop
git checkout develop
git merge version-1.0
git push origin develop

# When ready for production, merge develop to main
git checkout main
git merge develop
git push origin main
```
